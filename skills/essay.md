# Essay Skill SOP

本 SOP 指导 Coder agent（兼写作）在完成 essay 类作业时的论证组织 / 引用规范 / 学术诚信。
ground truth 全部从 task context（intake_result + user_constraints + 用户提供的阅读材料）拿；
不允许凭空引用不存在的文献、不允许造数据。

## 1. 读懂题面（先做这一步）

抽出：
- 字数下限 / 上限（如"不少于 1500 字"、"不超过 3 页"）
- 主题词 / 题眼（如"分析 X 的 Y 影响"——核心是"分析"，不是"介绍"）
- 必要章节（如"必须含引言 / 背景 / 论证 / 结论"）
- 立场要求（如"任选一方论述"、"中立分析"——决定是否需要驳论）
- 引用规范（如"必须引用至少 3 篇文献"、"APA / GB-7714"）

不确定时：
- 阅读材料看不懂的地方调 sandbox_convert_to_markdown 把 PDF/DOCX 转 markdown 后再读
- 写作风格优先看 profile.preferences.writing_style（formality / avg_sentence_len）

## 2. 立结构（写之前先列大纲）

不要直接开写。先在脑里 / 草稿区列一个 N 级提纲：

1. 引言（why this 题：为什么这个问题值得论述；用 1-2 句钩子）
2. 背景 / 概念界定（定义关键术语，避免后续歧义）
3. 论点 1 + 论据 + 论证（论据须可溯源，注明来源；论证用因果 / 对比 / 类比）
4. 论点 2 + 论据 + 论证
5. （可选）反方观点 / 驳论（提高论证完整度）
6. 结论（不只是重复论点；说明对实践 / 后续研究的启示）
7. 参考文献（按要求格式）

每个论点配 1-3 条论据；论据 ≥ 2 条比 1 条更稳。

## 3. 写作风格（按 profile.preferences.writing_style）

formality（正式度）：
- low：可用「我觉得」「这事儿」等口语；句末偶尔感叹号
- medium：标准学术中文；不用口语；不用感叹号
- high：偏学术论文；多被动 / 名词化；引文比例高

avg_sentence_len（平均句长，中文字符数）：
- 25 左右是一般偏长；想写得更易读用 18-20；想体现深度用 30+

不要为了凑字数注水。每段 3-5 句，每句 1 个核心意思；散文化、不堆砌"首先 / 其次 / 再者"。

## 4. 引用规范（学术诚信关键）

任何引用外部资料（教材 / 论文 / 网页 / 维基）必须：
- 用引号 + 注明来源：`"原文" — 作者(年份)《标题》` 或 `[1] 作者. 标题. 期刊, 年份.`
- 单条直接引用 ≤ 30 字（超过用间接引用 + 自己语言转述 + 标来源）
- 参考文献章节列出全部来源；正文 [1][2] 和参考文献条目要对得上

间接引用（用自己语言转述别人观点）也要在句末标 `(作者, 年份)`，不能闷声塞进自己的话里。

## 5. 学术诚信（写作类硬要求）

参见 hwHandler 全局学术诚信约束（agents/coder.py 内 _ACADEMIC_INTEGRITY_PROMPT 已注入）：
- 全部内容须本人创作；引用须标来源
- 题目原文可直接引用作分析对象
- 单条引用 ≤ 30 字
- 不抄袭他人论文 / 同学作业（即便借鉴方法也必须自己语言重述）
- 产物中不出现他人姓名 / 学号 / 隐私
- 不代写不买卖
- 末尾「使用工具说明」段标注 AI 辅助

## 6. 自检 checklist（写完后逐条对照）

- [ ] 字数在 README 要求区间内
- [ ] 全部章节齐（引言 / 论证 / 结论 / 参考文献至少这 4 项）
- [ ] 每个论点都有 ≥ 1 条可溯源论据
- [ ] 引用都用了引号 + 来源；单条 ≤ 30 字
- [ ] 没有口语化句子（除非 formality=low）
- [ ] 没有出现"我觉得 / 我认为"过多堆叠（学术写作偶尔可，过多伤说服力）
- [ ] 末尾有「使用工具说明」段

## 7. 与 coding / lab_report skill 的边界（各司其职）

- 题面要求"实现某算法 / 跑实验" → 不是 essay，是 coding 或 lab_report；让 Planner 重选
- 题面要求"分析某算法 / 论述某理论" → 是 essay
- 题面要求"读一本书写读后感 + 不少于 X 字" → 是 essay
- 题面要求"对比 A / B 两种方法（无需实现）" → 是 essay（纯论述）；带"用代码实现并对比"才是 coding

不要在 essay 产物里塞代码块；解释算法用文字描述步骤即可，不用 ```python```。

## 异常处理速查

| 失败模式 | 应对 |
|---|---|
| 字数严重不足 | 先检查论据是否每条都展开了 1-2 句论证；不要凑词或重复 |
| 没有可引用文献 | 不要编造；写「无外部引用，仅基于 README 题面 + 教材 X 论述」并在 SUMMARY 第 7 章标注 |
| 用户给的阅读材料是 PDF | 调 sandbox_convert_to_markdown 转 markdown 后再读；引用要带原文页码 |
| 立场矛盾要求（如要中立又要表态） | SUMMARY 第 7 章「后续待办」标注矛盾，按用户对话补充意图来定，必要时 fall back 到中立分析 |
